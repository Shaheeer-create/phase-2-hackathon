# spec-validator

## Purpose
When the user provides a specification or asks to validate one, use this skill to review and validate the specification.

## Instructions for Agent
1. Read the provided specification carefully
2. Check for completeness:
   - Are all required sections present?
   - Are functional requirements clearly defined?
   - Are non-functional requirements specified?
   - Are acceptance criteria measurable?
3. Verify consistency:
   - Do requirements conflict with each other?
   - Are terms used consistently throughout?
4. Assess feasibility:
   - Are technical requirements realistic?
   - Are dependencies clearly defined?
   - Are success metrics achievable?
5. Identify risks:
   - Are there ambiguous requirements?
   - Are there missing edge cases?
   - Are there potential implementation challenges?
6. Provide feedback to the user with specific suggestions for improvement
7. Confirm when the specification meets quality standards

## When to Use
- User provides a specification for review
- User asks to validate requirements
- User wants feedback on a specification document
- Quality check is needed before implementation
