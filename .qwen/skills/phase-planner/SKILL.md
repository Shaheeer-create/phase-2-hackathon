# phase-planner

## Purpose
When the user needs to plan a project or break down a complex task, use this skill to create a phased approach with milestones.

## Instructions for Agent
1. Analyze the overall project or task to understand scope and requirements
2. Identify major deliverables and components
3. Group related tasks into logical phases with these considerations:
   - Dependencies between components
   - Risk mitigation (earlier phases should address high-risk items)
   - Value delivery (earlier phases should provide some value)
   - Resource availability and constraints
4. Define clear entry and exit criteria for each phase
5. Establish measurable milestones for each phase
6. Estimate timeline for each phase considering:
   - Complexity of tasks
   - Dependencies
   - Available resources
   - Potential risks and buffers
7. Identify potential risks for each phase and mitigation strategies
8. Present the phased plan to the user for review
9. Revise based on feedback

## Phase Planning Guidelines
- Each phase should have a clear, focused objective
- Phases should build upon each other
- Include integration points between phases
- Plan for testing and validation within each phase
- Consider rollback strategies if needed
- Account for stakeholder review and approval points

## When to Use
- User asks to plan a project or feature
- User needs to break down a complex task
- User wants to establish milestones and timelines
- Need to organize work for better management
