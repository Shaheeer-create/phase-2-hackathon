# spec-writer

## Purpose
When the user requests to create a specification, use this skill to generate a detailed specification document.

## Instructions for Agent
1. Analyze the user's request to understand the feature or task that needs specification
2. Gather requirements by asking clarifying questions if needed
3. Create a specification document with the following sections:
   - Title and description
   - Functional requirements
   - Non-functional requirements
   - User stories or use cases
   - Acceptance criteria
   - Technical constraints
   - Dependencies
   - Success metrics
4. Present the specification to the user for review
5. Revise based on feedback

## When to Use
- User asks to create a specification
- User wants to define requirements for a feature
- User needs a detailed document outlining what needs to be built
